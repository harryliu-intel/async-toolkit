Help Page

Introduction

This documents provides general help on the using and interrpreting the output of the Regression Test Engine; by provides an overview of the html layout the directory struture of the output. The user needs an frame enhanced browser in order to view the test results.

How to Run the Regression Test Engine

Please refer to the man page for the rte more details or type 'rte --help' for a simplified help. Please send bug reports to abe@fulcrummicro.com or enter them in Bugzilla.

How Regression Test Results Are Organized

After the all the tests are complete, a `results.html` page will we be generated and place in the top level directory specified for the test either on the command line or in the $RTEROOT environment variable. This page provides the links to navigating the remaining results of the particular test run. This page contains three frames
  • All tests frame: this frame contains a list of all the top level cells specified in the input cast. Left clicking on this link will provide the hierarchy of the cell in the main frame.
  • All cells frame: this frame contains a list of all the unique cells that were individually simulated in the regression test. They are color coded to indicate which tests passed or failed. Left clicking on a cell will send to you to a page containing the actual results of the test run.
  • Main Frame: this frame is used to display a different kinds of information:
    • The results of simulatings a given cell, including coverage page. A link to the actual cell definition in the users home directory.
    • Hierarchy page for all top level cells.
    • Test info page: this page provides information about the kinds of tests run by the rte and how to properly set up a cast cell to be properly run by this utility.
    • A statistics page providing some relevant test statistics.
    • This help page.

Results directory organization

The section presents an organization of the final output produced by the rte and describes the supporting files generated to enhance the overall output of this utility to make the environment more user friendly and informative. Note that all the specified directories are relative to the $RTEROOT environment variable.

index.html: One file that creates the HTML frames for display. This is the file you load to display the front page with frames. This file itself contains no text content.

help-doc.html: A help page provides general help on the using and interrpreting the output of the Regression Test Engine and duplicates some of the content in the man page.

test-info.html: This page describes the different categories of tests the Regression Test Engine performs and lists and describes the different directives. It also provides example of CAST cells with the various block filed in.

stylesheet.css: style sheet file hat controls a limited amount of color, font family, font size, font style, and positioning on the generated pages.

logs/$machine.log: Logs generated by running the client on a specific machine. It meant to allow the user track where cells are being simulated and is also useful for debugging purposes.

hier/top-level-cell.html: The tests directory contains the hierchy listing for the different top level cells specified in the input cast file.

modules/.../cellname.html: The simulatition files are written out according the actual modules they belong to. For instance BUF_1of4 in the module lib.buffer.half is written out to ./modules/lib/buffer/half/BUF_1of4.html.
