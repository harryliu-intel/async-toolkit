Diretive Support in LVE

The notion of directives in CAST provides a mechanism for annotating information about how a particular circuit should be synthesized, simulated or how these simulation results should be interperrated. Directives associated with the LVE fall in two broad categories. The first class of directive are not directly understood by the LVE but instead influence how the todo file for the LVE is generated using cast_query. Two directives fall in this category -- aspice_ignoreandrte_ignore. When these directives are specified in a given environment that environment is omitted from the generated todo file, if the directive is specified in the CELL level the cell is emitted with no environments, even if these directives are also specified in the cells environment and set to false. Note that this is a special case handling provided in cast_query.
